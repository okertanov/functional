<h5>Краткая компиляция irc-чата в рамках Ubuntu Open Week на тему "An Intrepid journey in Ubuntu Server land"</h5>
    <a href="https://wiki.ubuntu.com/MeetingLogs/openweekintrepid/IntrepidServer">Поход Убунту на серверы</a>. Докладчик: <a href="https://launchpad.net/~mathiaz">Mathias Gug</a>

Сам докладчик, как и некоторые представленные им разработчики, входят в команду <a href="https://wiki.ubuntu.com/ServerTeam">Ubuntu Server Team</a>.
Это группа людей, имеющих свои интересы как в серверном ПО, так и в сфере систем для корпоративного сектора: LDAP, AD, Web, почта, сетевая аутентификация.
<habracut />
За последний релиз-цикл (в рамках выпуска Intrepid) было достигнуто:
<ul>
    <li>    шифрование ~/Private с применением ecryptfs. Дальнейшим развитием будет создание графического интерфейса для управления зашифрованными каталогами.</li>
    <li>    был переписан ubuntu-vm-builder для поддержки плагинов, что позволило поддерживать новые релизы, шаблоны fstab, несколько гипервизоров.</li>
    <li>    добавлена опция загрузки с поврежденного(degraded) RAID-массива, которую давно ждали.</li>
    <li>    продаланна значительная работа по добавлению в основной репозиторий для spamassassin и clamav.</li>
    <li>      теперь их можно с легкостью подключить к postfix с помощью amavisd-new. Цель на будущее - заскриптовать установку этого всего хозяйства для достижения развертывания интегрированного рабочего окружения mail-сервера без каких-либо правок конфигов руками.</li>
    <li>    идут работы по добавлению поддержки cn=config в slapd. Гарантирована миграция slapd.conf в новую систему при апдейте в Intrepid. Хвалят этот новый бэкенд за простоту загрузки новых схем и доступу к ldap для приложений.</li>
    <li>    <a href="https://wiki.ubuntu.com/ServerTeam/KnowledgeBase#Ubuntu%20Server%20Guide ">Ubuntu Server Guide</a> обновился.</li>
</ul>
Далее идет рассказ о том, как много внимания они уделяют ошибкам: поиск, трекинг, исправление, тестирование, пакетирование.  Для желающих помочь существует <a href="https://bugs.launchpad.net/~ubuntu-server/+mentoring ">список тривиальных багов</a>, с них стоит начинать желающим приобщиться к <a href="https://wiki.ubuntu.com/MOTU">MOTU</a>.
Еще одним несложным способом помощи серверной команде является простая загрузка типовой поставки
(а возможно и альф/бет следующих релизов) на доступном вам серверном железе. Отчет об успешной или неуспешной установке на доступном серверном железе им тоже ценен. И наконец, вычитка материалов вики, Server Guide, заполнение <a href="https://wiki.ubuntu.com/ServerTeam/IdeaPool">пула идей</a> обычными пользователями для них представляют большой интерес. Видно, что помочь проекту можно многими путями во многих направлениях.  Постоянно просят помощи, а что же они делают сами в это время и насколько эффективна их работа? Пожалуйста:
<ul>
    <li>    идет постоянная фиксация проделанной работы в <a href="https://wiki.ubuntu.com/ServerTeam/Roadmap">Roadmap</a>.</li>
    <li>    встречи раз в неделю для дискуссий по насущным вопросам.</li>
    <li>    для координации задействован mailing-list и блог.</li>
</ul>
А теперь - слайды, то есть вопросы. Про RAID. Насколько соизмерим уровень поддержки этой технологии хранения в Убунту и Ред Хате? Ответ: Абсолютно такая же как и в Ред Хат.
Про tacacs+. Как оказалось, его нету в Убунту. Как добавить? Ответ: <a href="https://wiki.ubuntu.com/UbuntuDevelopment/NewPackages">wiki.ubuntu.com/UbuntuDevelopment/NewPackages</a>
